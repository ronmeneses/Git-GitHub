GIT & GITHUB

Crear llave SSH Publica y Privada

1.- ssh-keygen -t rsa -b 4096 -C "email"
2.- Preguntara el nombre y la direcion de donde la quieres guardar (solo dale Enter).
3.- El passphrase es poner una contraseña a la llave una medida de seguridad extra, si gustas lo pones pero si no solo le das Enter.
4.- Confirmas la contraseña del passphrase o le das Enter si no pusiste nada.
De esa manera generaste la llave publica y privada.

Agregar la llave anuestro entorno (S.O.)

Para Windows y Linux
1.- Verificamos si el servidor de ssh esta corriendo con eval $(ssh-agent -s) si te regresa un pid el serviro se esta ejecutando.
2.- Agregamos la llave privada ssh-add .ssh/id_rsa.
Para Mac
1.- Verificamos si el servidor de ssh esta corriendo con eval "$(ssh-agent -s)" si te regresa un pid el serviro se esta ejecutando.
2.- Creamos el archivo config en la carpeta .ssh con el siguiente contenido si es que el archivno existe. 

Host *
                           AddKeysToAgent yes
                        UseKeychain yes
                                   IdentityFile ~/.ssh/id_rsa
3.- Agregamos la llave privada con ssh-add -K ~/.ssh/id_rsa si encaso te da algun error solo borra -K y listo.

Tag y versiones en Git y GitHu

1.- Ver los tag --> git tag
2.- Crear un tag con el comando git tag -a nombreDeTag idCommitDeDondeQuieresCrearElTag -m "mensaje que tendra el tag"
3.- Para poder ver a que commit estan asignados los tags --> git show-ref --tags.
4.- Enviar los tags o tag al repositorio en GitHub. Antes de reaizar cualquier push100pre realiza antes un pull. 

4.1.- git pull origin master
                    Para enviar un solo tag
                        4.2.- git push origin nombreDelTag
                    Para enviar todos los tags
                        4.3.- git push origin --tags
5.- Eliminamos el tag de forma local --> git tag -d nombreDeTag.
6.- Si el tag fue subido a nuestro repositorio se elimina --> git push origin :refs/tags/nombreDelTag. el comando no borra el tag en local.

Realizar contribuciones a proyectos open source

1.- Primero debes de realizar el Fork del poyecto al cual quieras contribuir, esto crear un repositorio con el mismo nobre pero en tu perfil.
2.- Decargamos el proyecto desde tu perfil a tu local (computadora) para que puedas realizar tus cambios con --> git clone linkDelProyecto.
3.- Una vez realizado tus commits el push que realiaces sera solo al de tu perfil (el Fork).
4.- GitHub sabe que tu repositorio es un fork de otro proyecto, y para que tus cambios sean aceptados debes realizar un Pull Request al proyecto original y esperar quesea aceptado.
5.- Pero si el dueño del repositorio original tambien trabajo en su repositorio entonces tendremos un problema ya q nosotros no tendremos esos cambios. Y GitHub te notificara que te faltan los cambios del repositorio original
6.- Y para que tu puedas tener los cambios del repositorio original solo tienes que hacer un pull request de los cambios en tu perfil y listo.
7.- Pero hay otra manera el cual es de pros :) y eso es por consola con los siguietes comandos. 

7.1.- Actualmente tenemos el remoto origin apuntando nuestro repositorio
                                pero necesitamos otro remoto apuntando al link del repo original.
                                Que normalmente tiene el nombre de upstream, y el comando es el siguiente.
                                --> git remote add upstream https://LinkDelRepositorioOriginal.
                         7.2.- Ahora solo necesitamos traer los cambios del repo original con un pull con el nuevo remoto.
                                --> git pull upstream master
                         7.3.- EL comando anterio nos trajo los commits del repo original ahora solo resta enviar esos commits
                               a nuestro repositorios (Fork) a esto se lo conoce como fusion (even).
                               --> git push origin master
